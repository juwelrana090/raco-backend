# S3 Module

## Purpose
AWS S3 file storage service for product image uploads with support for multiple S3-compatible providers.

## Architecture
- **Service Layer**: S3Service wraps AWS SDK v3 operations
- **Configurable Endpoint**: Supports AWS S3, DigitalOcean Spaces, MinIO, LocalStack
- **Path Style**: Configurable virtual-host or path-style addressing
- **CDN Support**: Returns CDN URLs if configured

## Key Features

### 1. Multi-Provider Support
- **AWS S3**: Default (uses virtual-host style)
- **DigitalOcean Spaces**: Set `AWS_ENDPOINT` to Spaces endpoint
- **MinIO**: Set `AWS_ENDPOINT` to MinIO endpoint + `AWS_USE_PATH_STYLE_ENDPOINT=true`
- **LocalStack**: Set `AWS_ENDPOINT=http://localhost:4566` + `AWS_USE_PATH_STYLE_ENDPOINT=true`

### 2. File Operations
- `uploadFile()`: Generic file upload with folder support
- `deleteFile()`: Delete file by S3 key
- `uploadProductImage()`: Product-specific upload with `products/{productId}/` prefix
- `extractKeyFromUrl()`: Parse S3 key from CDN/S3 URL

### 3. Security & Validation
- **Public-read ACL**: Files are publicly accessible via URL
- **UUID Filenames**: Prevents naming collisions
- **Key Pattern**: `products/{folder}/{uuid}.{ext}`
- **Credential Safety**: Never logs AWS credentials

### 4. Configuration via Environment Variables
```bash
AWS_BUCKET=your-bucket-name-here
AWS_DEFAULT_REGION=your-region-here
AWS_ACCESS_KEY_ID=your-access-key-id-here
AWS_SECRET_ACCESS_KEY=your-secret-access-key-here
AWS_ENDPOINT=your-s3-endpoint-here
AWS_CDN_URL=your-cdn-url-here
AWS_USE_PATH_STYLE_ENDPOINT=false
AWS_S3_MAX_FILE_SIZE=5242880
AWS_S3_ALLOWED_MIME_TYPES=image/jpeg,image/png,image/webp,image/gif
```

## AWS SDK v3 Configuration
Uses modular `@aws-sdk/client-s3` package (tree-shakeable):

```typescript
this.s3Client = new S3Client({
  region,
  credentials: {
    accessKeyId,
    secretAccessKey,
  },
  endpoint: endpoint || undefined,
  forcePathStyle: usePathStyle,
});
```

## Upload Process
1. Generate UUID-based filename: `${uuidv4()}.${ext}`
2. Create S3 key: `products/{folder}/{uniqueFileName}`
3. Upload with `PutObjectCommand`:
   - `Body`: file buffer
   - `ContentType`: MIME type from `getContentType()`
   - `ACL`: `public-read`
4. Return URL: CDN URL if available, otherwise S3 URL

## MIME Type Detection
`getContentType()` maps file extensions to MIME types:
- `.jpg`, `.jpeg` → `image/jpeg`
- `.png` → `image/png`
- `.gif` → `image/gif`
- `.webp` → `image/webp`
- fallback → `application/octet-stream`

## URL Patterns
- **With CDN**: `{AWS_CDN_URL}/{key}`
- **Without CDN**: `https://{bucket}.s3.{region}.amazonaws.com/{key}`

## Dependencies
- `@aws-sdk/client-s3`: AWS SDK v3 (already installed)
- `ConfigService`: NestJS configuration service
- `uuid`: UUID generation for unique filenames

## Integration with Products Module
- `ProductsService` injects `S3Service`
- Image upload endpoint: `POST /products/:id/image` (admin only)
- Image delete endpoint: `DELETE /products/:id/image` (admin only)
- Old image deleted before uploading new one
- Image deleted from S3 when product is deleted

## Error Handling
- S3 upload errors are logged and re-thrown with descriptive messages
- Delete failures are logged but don't block product deletion
- Missing credentials throw error during service initialization

## Local Development
Using LocalStack for local S3 mock:
```bash
# Start LocalStack
docker run -p 4566:4566 localstack/localstack

# Set environment variables
AWS_ENDPOINT=http://localhost:4566
AWS_USE_PATH_STYLE_ENDPOINT=true
```
